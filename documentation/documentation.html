VORTEX DOCUMENTATION

Complete setup and customization guide for the Vortex E-Sports HTML template. No prior experience beyond basic HTML editing required.

01 Introduction

Vortex is a premium dark HTML template for professional esports organizations, gaming teams, streamers, and content creators. It features aggressive display typography, vibrant orange accents, a real-time countdown timer, animated player roster cards, and a complete match schedule system.

Sections Included

  • Hero — Full-height with live countdown timer and animated hero stats
  • Games Bar — Switchable active game tags
  • Roster — 5 player cards with hexagonal avatars, KPI stats, role badges and social links
  • Match Schedule — Upcoming matches and past results with VOD links
  • Hall of Fame — Trophy cards with Gold, Silver, Bronze tiers
  • Stream — Twitch player mockup UI and social feed posts
  • Sponsors — Partner grid with title and regular tiers
  • Join / Tryouts — Application form with game selector
  • Footer — Full sitemap with social icons

02 Requirements

Vortex is pure HTML/CSS/JavaScript. No build tools, no npm, no Node.js required.

  • Any modern browser (Chrome, Firefox, Safari, Edge)
  • A text editor (VS Code recommended)
  • Any static web host for deployment (GitHub Pages, Netlify, Vercel, shared hosting)
⚡ Open index.html directly in your browser to preview — everything works immediately, no server needed.

04 Quick Start

1. Preview locally

Open docs/index.html in any browser. All animations, fonts and CDN scripts load automatically.

2. Set your team name

Search for VORTEX in docs/index.html and replace with your team name. The name appears in the nav logo, hero, join section, and footer.

3. Deploy

Upload the three files from docs/ to any static host. For GitHub Pages: push to a repo, enable Pages under Settings → Pages → Branch: main → / (root).

ℹ Keep docs/index.html, style.css and script.js in the same folder. HTML references the other files with relative paths.

05 Colors & Theme

All colors are CSS custom properties in style.css. Change the brand color in one place and the entire template updates.

/* style.css — :root block */
:root {
  /* Backgrounds */
  --bg: #0a0a0c;        /* Page background */
  --bg2: #111116;       /* Cards, sections */
  --bg3: #18181f;       /* Deep elements, inputs */

  /* Brand / Accent */
  --orange: #ff4b2b;    /* Primary accent */
  --orange2: #ff8c00;   /* Gradient end */
  --orange-dim: rgba(255,75,43,0.12);   /* Tinted backgrounds */
  --orange-border: rgba(255,75,43,0.3); /* Glowing borders */

  /* Text */
  --text: #c8c8d8;
  --text-muted: #5a5a72;
  --white: #f0f0ff;
}

Example: Purple/Blue gaming theme

:root {
  --orange: #8b5cf6;    /* Purple */
  --orange2: #3b82f6;   /* Blue */
  --orange-dim: rgba(139,92,246,0.12);
  --orange-border: rgba(139,92,246,0.3);
}
⚡ When changing --orange, always update --orange-dim and --orange-border using the same RGB values with lower opacity. This keeps the glow effects consistent.

07 Countdown Timer

The countdown in the hero counts down to a real date and updates every second. To change the target date, edit one line in script.js:

// script.js — line ~20
// Format: 'YYYY-MM-DDTHH:MM:SS'
const targetDate = new Date('2025-04-26T18:00:00');
ℹ The countdown uses the visitor's local time zone. If your event is in a specific time zone, append the offset: '2025-06-01T20:00:00-05:00' for UTC-5.

Update the event label

The event name below the countdown is a plain text span in index.html:

<span class="countdown-event">APEX WORLD SERIES — GRAND FINAL</span>

08 Roster Cards

Each player card is a self-contained block in the .roster-grid div. To update a player:

<div class="player-card">
  <!-- 1. Accent bar color (change --c to any hex) -->
  <div class="player-accent" style="--c:#ff4b2b"></div>

  <!-- 2. Hex avatar initials and stroke color -->
  <text fill="rgba(255,75,43,0.35)" ...>SH</text>

  <!-- 3. Role tag -->
  <span class="player-role">#FRAGGER</span>

  <!-- 4. Alias and real name -->
  <h3>SHADOWBYTE</h3>
  <span class="player-realname">Marcus Reyes</span>

  <!-- 5. KPI stats -->
  <div class="player-kpis">
    <div><span>KDA</...><strong>8.4</...></div>
    ...
  </div>
</div>

Featured / Captain card

Add the class featured to a player card to give it the glowing border and captain ribbon:

<div class="player-card featured">
  <div class="player-ribbon">CAPTAIN</div>
  ...

3D Tilt effect

The card tilt on hover is handled automatically by JavaScript — no configuration needed. To disable it, remove the player-card tilt event listeners block in script.js.

09 Match Schedule

Each match is a .match-row div. Status classes control the badge color:

ClassColorUse for
.status.upcomingOrangeFuture matches
.status.winGreenWon matches
.status.lossRed (add to CSS)Lost matches

Add the class past to a .match-row to visually dim it (opacity 0.65) for past results:

<div class="match-row past"> <!-- Past match -->
<div class="match-row">       <!-- Upcoming match -->

Adding a loss status

The template includes win and upcoming styles. To add a loss style, paste this into style.css:

.status.loss {
  background: rgba(239,68,68,0.12);
  color: #f87171;
  border: 1px solid rgba(239,68,68,0.3);
}

10 Animations

Scroll animations use AOS. Add data-aos to any element to animate it on scroll:

data-aos="fade-up"          // Most common
data-aos="fade-right"       // Slides from left
data-aos="zoom-in"          // Scales in
data-aos-delay="200"       // ms delay
data-aos-duration="800"    // ms duration

CSS-only animations (always run)

  • Cursor glow — follows mouse via JS mousemove event
  • Live badge pulse — @keyframes blink-dot
  • Stream scanline — @keyframes stream-scan
  • Player card 3D tilt — JS mousemove on .player-card

Disable cursor glow

/* Set display none in style.css */
.cursor-glow { display: none; }
